`/mmr` merges an existing PR/MR with squash commits and deletes the source branch afterward. Use it when your PR/MR has been approved and is ready to land -- it handles the squash merge and cleanup in one step. It works with both GitHub PRs and GitLab MRs. Usually you will use `/scpmmr` for the full pipeline, but `/mmr` is useful when the PR/MR already exists and just needs merging. For the commit workflow walkthrough, run `/ccwork tour workflow`.

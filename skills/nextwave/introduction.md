`/nextwave` executes one pending wave end-to-end using the Orchestrator/Prime/Flight protocol on a filesystem message bus. The Orchestrator (top-level session) drives the loop; a Prime sub-agent plans the wave and owns post-flight merges; Flight sub-agents implement one issue each in isolated worktrees. Flights within a flight-group spawn in a single parallel tool-use block; flights themselves are sequential. Two modes: `/nextwave` (interactive, with approval gate before anything pushes) and `/nextwave auto` (no gate — used by `/wavemachine`). For the full walkthrough, run `/ccwork tour`.

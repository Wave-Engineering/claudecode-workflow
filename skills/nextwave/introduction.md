`/nextwave` executes the next pending wave of parallel sub-agents on isolated worktrees, using flight-based conflict avoidance to prevent file-level clashes. Use it after `/prepwaves` has planned the execution order -- it spawns agents for each issue in the wave, partitioned into safe parallel flights. Each agent works on its own worktree and merges via PR/MR. For a full walkthrough of parallel execution, run `/ccwork tour`.

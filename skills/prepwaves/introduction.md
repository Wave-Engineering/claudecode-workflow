`/prepwaves` analyzes a master issue and its sub-issues, validates their specs for agent readiness, and computes dependency-ordered waves for parallel execution. Use it after `/assesswaves` confirms the work is suitable for parallelism -- it produces the wave plan that `/nextwave` consumes. It checks that every sub-issue has clear acceptance criteria, implementation steps, and no circular dependencies. For a full walkthrough, run `/ccwork tour`.

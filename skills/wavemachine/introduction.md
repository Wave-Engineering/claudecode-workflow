`/wavemachine` is the Orchestrator-level autopilot for a multi-wave plan. It runs a simple loop in the top-level session: check `wave_health_check` as a circuit breaker, ask `wave_next_pending` for the next wave, delegate that single wave to `/nextwave auto`, parse the canonical status JSON from the result, and either loop back or exit. There is no background worker — the loop lives here, at the top level, where the `Agent` tool exists so `/nextwave auto` can spawn its parallel Flights correctly (this was v1's broken shape: a backgrounded Agent child lacked Agent itself and silently collapsed every wave to serial). The loop exits on a non-HEALTHY health check, on `wave_next_pending` returning null (plan complete), on `/nextwave auto` returning BLOCKED or FAIL, or on user interrupt. The `wavemachine_active` flag in `.claude/status/state.json` (the statusline 🌊 signal) is set on entry via `wave-status wavemachine-start` and cleared on EVERY exit path via `wave-status wavemachine-stop` — no exception. The in-flight wave's bus tree is preserved on abort for forensics. For the full walkthrough, run `/ccwork tour`.
